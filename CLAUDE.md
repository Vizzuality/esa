## Project Overview

ESA is a geospatial data visualization platform for Earth observation data. It consists of a monorepo with three main workspaces: a Next.js frontend (`client`), a Strapi CMS backend (`cms`), and a Python data processing pipeline (`data-processing`).

## Commands

All commands are run from the repo root using Yarn workspaces.

### Development

```bash
yarn install                  # Install all workspace dependencies
yarn client dev               # Next.js dev server at http://localhost:3000
yarn cms dev                  # Strapi dev server at http://localhost:1337/admin
docker-compose up --build     # All-in-one: db (PostGIS), CMS, client
```

### Build & Type Checking

```bash
yarn client build             # Next.js production build
yarn client check-types       # TypeScript type checking (tsc)
yarn client lint              # ESLint
yarn types build              # Regenerate TypeScript types from Strapi OpenAPI schema
```

### Testing

```bash
yarn client test                              # Run all Playwright E2E tests
yarn client playwright install --with-deps    # Install test browsers (first time)
```

There is no unit test framework — only Playwright E2E tests in `client/`.

### CMS

```bash
yarn cms build                # Build Strapi admin panel
yarn cms seed                 # Import seed data: strapi import -f ./seed.tar.gz
```

## Architecture

### Monorepo Structure

```
client/          # Next.js 16 frontend (App Router)
cms/             # Strapi 4 headless CMS + PostgreSQL/PostGIS
data-processing/ # Python geospatial data pipeline (Conda/Jupyter)
types/           # Orval-generated TypeScript types from Strapi OpenAPI
infrastructure/  # Terraform (AWS Elastic Beanstalk, ECR, S3)
```

### Frontend (`client/src`)

The client uses **Next.js App Router** with server components as the default. Key patterns:

- **Containers vs Components**: `containers/` are smart components that manage state, fetch data, and coordinate child components. `components/` are presentational.
- **State management**: [Jotai](https://jotai.org/) atoms in `store/` for client state. URL query params synced via `nuqs` for shareable/navigable state (step, category, search filters).
- **Data fetching**: TanStack Query (React Query v4). Pages prefetch server-side with `dehydrate`/`HydrationBoundary`. Client components use auto-generated hooks (e.g., `useGetStories()`).
- **API types**: Auto-generated by Orval from Strapi's OpenAPI endpoint. Run `yarn types build` after CMS model changes. Generated files live in `client/src/types/generated/`.
- **Maps**: React Map GL + Mapbox GL 3 for base maps; Deck.gl 9 for advanced geospatial layers. Map state (viewport, layers, settings) lives in `store/map.ts`.
- **Styling**: Tailwind CSS v3 with CSS variables for theming. Radix UI primitives + class-variance-authority for component variants.
- **Environment**: Type-safe env vars via `@t3-oss/env-nextjs` in `env.mjs`. See `client/README.md` for the full list of required `NEXT_PUBLIC_*` vars (Mapbox token is required).

### Key Routing

- `/home` — landing page (redirect from `/`)
- `/globe` — main interactive globe/map view (`containers/globe/`)
- `/stories/[id]` — story detail with scroll-driven step navigation (`containers/story/`)

The story viewer syncs a `?step=N` query param and switches between globe/mercator projections: globe projection on step 1 and the last step, mercator in between.

### CMS → Client Data Flow

1. Client fetches content/layers/datasets from `NEXT_PUBLIC_API_URL` (Strapi REST API)
2. Strapi serves from PostgreSQL with PostGIS
3. Media assets served from AWS S3

Key Strapi collections: `layers`, `datasets`, `dataset-groups`, `stories`, `top-stories`, `tags`, `categories`, `ifis`.

### Data Processing

Python pipeline in `data-processing/` uses Conda/Mamba environment (`environment.yml`) with geospatial packages (GDAL, Rasterio, Fiona). Run via Jupyter Lab or Docker. Processes raster/vector data for visualization layers.

## Environment Setup

Node version is pinned in `.nvmrc` (`22.11.0`). Use `nvm use` before installing.

Client requires `.env.local` with at minimum:
- `NEXT_PUBLIC_API_URL` — Strapi API endpoint
- `NEXT_PUBLIC_MAPBOX_API_TOKEN` — required for maps to render

CMS requires `.env` with database credentials, S3 config, and Strapi security keys. See `cms/README.md` for the full list.
