## Architecture

Next.js 16 App Router with three routes under the `(landing)` route group:

- `/home` — Landing page with satellite interactive
- `/globe` — Story map explorer with filters, search, category browsing
- `/stories/[id]` — Step-by-step story viewer with synchronized map

Root `/` redirects to `/globe` (returning visitors via `esa-visited` cookie) or `/home` (new visitors).

### Container/Component Split

- `src/containers/` — Smart components that wire up state, hooks, and API calls (story viewer, globe explorer, map orchestration)
- `src/components/ui/` — Radix UI / shadcn primitives (card, dialog, slider, tooltip, etc.)
- `src/components/map/` — Map-specific UI: legend (gradient, choropleth, timeline, matrix types), layer rendering, controls
- `src/components/chart/` — Chart.js visualizations

### State Management (Three Systems)

1. **Jotai atoms** (`src/store/`) — Ephemeral UI state: map viewport (`bboxAtom`), active layers (`layersAtom`), layer settings, popups, markers
2. **NUQS** (`src/store/globe.ts`, `src/store/stories.ts`) — URL-synced state: category, search, tags/ifis/status filters, story step. Makes filter state shareable via URL query params
3. **React Query v4** — Server state: all CMS data fetching via Orval-generated hooks (`useGetStories`, `useGetCategories`, etc.)

### Map Stack

- **Mapbox GL 3.12** via react-map-gl — Base map rendering, globe/mercator projection switching
- **Deck.gl 9.2** — Advanced overlay layers (aggregation, geo, mesh)
- **Layer types:** Mapbox native layers, Deck.gl layers, animated tile layers (APNG), markers
- Map constants in `src/constants/map.ts` (default viewport, bbox, style URL)

### Story System

Each story has ordered steps. A step can include: text content, map configuration (bbox, bearing, pitch, layers), clickable markers with media, and image/video carousels. Step navigation is URL-synced via `useSyncStep()`. Changing step triggers map fly-to and layer updates.

## Key Patterns

### SSR Prefetch + Hydration

Page components use `getQueryClient().prefetchQuery()` server-side, then `dehydrate`/`HydrationBoundary` to pass data to client. Client hooks (`useGetStoriesId`, etc.) pick up hydrated cache.

### Generated API Types

Types in `src/types/generated/` are auto-generated by Orval from the Strapi OpenAPI spec. Do not edit these files. To regenerate: `yarn types build` from monorepo root. The generated hooks use the Axios instance from `src/services/api/index.ts` as a mutator.

### Import Order (ESLint-enforced)

Strict grouping: externals → `@/env` → `@/lib` → `@/data` → `@/store` → `@/types` → `@/app` → `@/hooks` → `@/constants` → `@/containers` → `@/components` → `@/services`

### Environment Variables

Validated at build time via T3 Env (`src/env.mjs`). Client-side vars prefixed `NEXT_PUBLIC_`. Server-side: `GDA_MASTER_DATA_FUNCTION_BASE_URL`, `GDA_MASTER_DATA_FUNCTION_KEY` (Azure Function proxy).

## Path Alias

`@/*` maps to `./src/*` (configured in tsconfig.json).

## Styling

Tailwind CSS 3 with HSL CSS variables for design tokens (defined in `src/styles/globals.css`). Class-based dark mode. Uses `cn()` utility from `src/lib/classnames/` (clsx + tailwind-merge). Component variants via class-variance-authority. Prettier plugin sorts Tailwind classes.

## SVG Handling

SVGs imported as React components via SVGR (configured in next.config.mjs webpack/turbopack rules).
